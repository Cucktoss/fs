# u235fs — учебная файловая система распада урана-235

`u235fs` — это **загружаемый модуль ядра Linux** (`.ko`), который регистрирует
собственную монтируемую in-memory pseudo-файловую систему. После монтирования
пользователь взаимодействует с упрощённой моделью радиоактивного распада U-235
через виртуальные файлы внутри точки монтирования.

Это не FUSE, не userspace-программа и не имитация через procfs/sysfs: ФС
регистрируется через `register_filesystem()` и монтируется командой
`mount -t u235fs ...`.

---

## 1. Что делает файловая система

После монтирования в корне сразу появляется файл `PARAMS` с параметрами модели.
Создание файла `RAM` запускает симуляцию: появляются файлы состояния
(`TEMPERATURE`, `DECAYED_NUCLEI`, `DECAY_RATE`, `ELAPSED_TIME`, `STATE`), которые
обновляются раз в секунду. Удаление `RAM` ставит симуляцию на паузу с сохранением
значений; повторное создание `RAM` продолжает её с того же места.

---

## 2. Требования

* Linux x86_64 (модель использует FPU; модуль рассчитан на x86_64);
* ядро Linux 5.15+ / 6.x;
* установленные kernel headers для текущего ядра;
* `build-essential`, `make`, `gcc`.

> Замечание про FPU: в ядре нет `libm`, а SSE/FP по умолчанию отключены. Поэтому
> `exp()` и форматирование чисел реализованы вручную, объект собирается с `-msse
> -msse2`, а все операции с плавающей точкой выполняются строго внутри
> `kernel_fpu_begin()/kernel_fpu_end()`.

### Установка зависимостей (Ubuntu/Debian)

```bash
sudo apt update
sudo apt install -y build-essential linux-headers-$(uname -r) make gcc
```

## 4. Запуск demo

```bash
chmod +x scripts/*.sh
sudo ./scripts/demo.sh
```

`demo.sh` собирает и загружает модуль, монтирует ФС, показывает `PARAMS`, меняет
параметры, запускает симуляцию, демонстрирует обновление состояния, паузу,
продолжение, проверяет блокировку `PARAMS`, затем размонтирует и выгружает модуль.

## 5. Запуск тестов

```bash
sudo ./scripts/test.sh
```

Скрипт проверяет сборку, загрузку, монтирование, наличие `PARAMS`,
редактируемость до запуска, появление файлов состояния, рост `ELAPSED_TIME` и
`DECAYED_NUCLEI`, вычисление `DECAY_RATE`/`TEMPERATURE`, паузу/продолжение,
блокировку `PARAMS` и корректную очистку. При ошибке завершается с ненулевым кодом.

## 6. Ручной сценарий

```bash
make
sudo insmod u235fs.ko
sudo mkdir -p /mnt/u235fs
sudo mount -t u235fs none /mnt/u235fs

ls -la /mnt/u235fs
cat /mnt/u235fs/PARAMS

# изменить параметры (только до создания RAM):
echo "mass=1500
volume=20
initial_frequency=2.5
impurities=3
external_neutrons=10
neutron_speed=2500" | sudo tee /mnt/u235fs/PARAMS

# запуск симуляции:
sudo touch /mnt/u235fs/RAM
sleep 2
cat /mnt/u235fs/STATE

# пауза:
sudo rm /mnt/u235fs/RAM
cat /mnt/u235fs/STATE     # state=paused, значения сохранены

# продолжение:
sudo touch /mnt/u235fs/RAM
cat /mnt/u235fs/STATE     # state=running, время продолжилось

# попытка изменить PARAMS после запуска -> ошибка (EPERM):
echo "mass=1" | sudo tee /mnt/u235fs/PARAMS

# завершение:
sudo umount /mnt/u235fs
sudo rmmod u235fs
```

## 7. Файлы внутри ФС

| Файл             | Назначение                                                       |
|------------------|------------------------------------------------------------------|
| `PARAMS`         | Параметры модели в формате `key=value`. Редактируем до `RAM`.    |
| `RAM`            | Управляющий файл. Его наличие = симуляция запущена.              |
| `TEMPERATURE`    | Условная температура (°C).                                       |
| `DECAYED_NUCLEI` | Количество распавшихся и оставшихся ядер.                       |
| `DECAY_RATE`     | Текущая скорость распада (ядер/с).                              |
| `ELAPSED_TIME`   | Прошедшее реальное и модельное время.                          |
| `STATE`          | Полная сводка состояния симуляции.                              |

---

## 8. Описание `PARAMS`

Значения по умолчанию:

```
mass=1000
volume=10
initial_frequency=1.0
impurities=0
external_neutrons=0
neutron_speed=2200
```

* `mass` — масса U-235 в граммах (влияет на начальное число ядер `N0`);
* `volume` — наблюдаемый объём (влияет на условную плотность/температуру);
* `initial_frequency` — множитель активности модели (≥ 0);
* `impurities` — количество примесей (снижают активность);
* `external_neutrons` — число сторонних нейтронов (добавляют индуцированную активность);
* `neutron_speed` — скорость сторонних нейтронов.

`PARAMS` можно изменять только **до** первого создания файла `RAM`. После запуска
запись в `PARAMS` возвращает `-EPERM`.

---

## 9. Логика `RAM`

* `touch RAM` — запускает симуляцию. При первом запуске блокируется `PARAMS`,
  вычисляется `N0` и создаются файлы состояния.
* `rm RAM` — ставит симуляцию на паузу. Таймер останавливается, но все значения
  (время, число ядер и т.д.) сохраняются в памяти модуля; `STATE` показывает
  `state=paused`, `ram_present=0`.
* повторный `touch RAM` — продолжает симуляцию с сохранённого состояния (не с нуля).

## 10. Математическая модель

Модель **учебная и упрощённая**. Она опирается на базовый закон радиоактивного
распада, но не претендует на физически точное моделирование реактора или цепной
реакции.

**Почему экспоненциальный закон.** Радиоактивный распад описывается уравнением

```
N(t) = N0 * exp(-lambda * t),   lambda = ln(2) / T_half
```

где `N0` — начальное число ядер, `N(t)` — число нераспавшихся ядер,
`lambda` — постоянная распада, `T_half` — период полураспада. Это естественная
и корректная база: число распадов в единицу времени пропорционально текущему
числу ядер.

**Зачем масштабирование времени.** Реальный период полураспада U-235 ≈ 7.04·10⁸
лет, поэтому за несколько секунд изменения были бы неразличимы. Введено
масштабирование: 1 реальная секунда симуляции = `1 000 000` модельных лет
(`MODEL_YEARS_PER_SECOND`). Так распад становится заметным за время демонстрации.

**Как масса влияет на число ядер.**

```
moles = mass / 235.0
N0    = moles * Avogadro      (Avogadro = 6.02214076e23)
```

Чем больше масса, тем больше `N0` и тем выше абсолютное число распадов.

**Как влияют примеси, объём и нейтроны (условные коэффициенты).**

```
impurity_factor = 1 / (1 + impurities * 0.05)              # примеси снижают активность
neutron_factor  = 1 + external_neutrons * neutron_speed * 1e-5   # нейтроны повышают (с верхним пределом)
density_factor  = mass / max(volume, 1)                    # меньший объём -> выше условная плотность
lambda_eff      = lambda * initial_frequency * impurity_factor * neutron_factor
```

Каждый тик (1 раз в секунду):

```
model_years   += MODEL_YEARS_PER_SECOND
remaining      = N0 * exp(-lambda_eff * model_years)
decayed        = N0 - remaining
decay_rate     = (remaining_prev - remaining) / 1s
temperature    = BASE_TEMPERATURE + decay_rate * TEMP_FACTOR * density_factor
```

Все значения детерминированы (никаких случайных чисел) и ограничены
(`clamp`): `initial_frequency ≥ 0`, `volume ≥ 1`, `mass ≥ 0`, `impurities ≥ 0`,
`external_neutrons ≥ 0`, `neutron_speed ≥ 0`, `impurity_factor > 0`,
`neutron_factor ≤ 10`, `0 ≤ decayed ≤ N0`, `remaining ≥ 0`,
`BASE_TEMPERATURE ≤ temperature ≤ 1500`.

**Почему модель учебная.** Используется только закон распада одиночного нуклида с
условными поправочными коэффициентами для наглядности влияния параметров. Цепная
реакция, нейтронный баланс, критичность и теплофизика реактора не моделируются.

---

## 11. Ограничения

* x86_64 only (используется kernel FPU для double-арифметики).
* In-memory: данные не сохраняются на диск и теряются при размонтировании.
* Поддерживается один типичный mount; создаётся/удаляется только файл `RAM`.
* `PARAMS` блокируется навсегда после первого старта (в рамках жизни этого mount).
* Модель упрощённая и не отражает реальную физику реактора.

## 12. Полная очистка после демонстрации

```bash
sudo ./scripts/clean.sh
```

Скрипт размонтирует ФС (если смонтирована), выгрузит модуль (если загружен),
удалит точку монтирования и временные файлы сборки. Он не падает, если что-то
уже размонтировано/выгружено.
